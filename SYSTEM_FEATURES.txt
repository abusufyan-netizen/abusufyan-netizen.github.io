1. CORE ARCHITECTURE (BACKEND & DATA LAYER)
----------------------------------------------------------------

[A] The Tools Registry (config/tools.yaml)
- Description: The single source of truth for every tool on the platform.
- Features:
    - Dynamic Metadata: Stores SEO titles, descriptions, and keywords.
    - Technical Specs: Defines input/output formats and processing types (client/server).
    - Release Management: Uses 'releaseDate' to handle scheduled deployments.
    - Priority System: Numerical weight (1-10) for sorting and visibility.
    - FAQ Schema: Stores tool-specific FAQs for Google Rich Snippets.

[B] The Parsing Engine (lib/tools.ts)
- Description: Bridges the YAML data to the Next.js frontend.
- Features:
    - Slug-to-Tool Mapping: Dynamic lookup of tool configurations.
    - Category Aggregation: Automatically compiles unique categories from the registry.
    - Safety Filters: Ensures "Coming Soon" or future-dated tools are hidden from users but visible in development.

[C] Technical SEO Layer (next.config.js & sitemap.ts)
- Description: Controls crawl behavior and URL health.
- Features:
    - Slash-Free URLs: Enforces standard slash-less paths for SEO consistency.
    - 301 Redirect Engine: Automates the transition from legacy /category/ paths to premium /hub/ paths.
    - Automated Sitemap: Dynamically generates URLs for 40+ tools, 90+ blog posts, and 8 Topical Hubs.
    - Security Headers: Hardened CSP, HSTS, and XSS protection.
